# 个人摄影作品分享网站搭建计划

## 1. 这次先做最小可执行版本

这次不追求一步到位。先做一个能上线、能展示、能继续加内容的版本。

这个版本只有 4 个页面：

```text
/                  首页
/gallery           作品列表页
/gallery/[slug]    单个作品系列页
/about             关于页
```

每个页面的作用如下：

| 页面 | 用途 | 大白话说明 |
| --- | --- | --- |
| 首页 | 放一张主照片、名字、简介、精选作品 | 访客一打开就知道你是谁，拍什么 |
| 作品列表页 | 展示所有作品系列 | 相当于总目录 |
| 单个作品系列页 | 展示某一组照片和说明文字 | 比如「香港街头 2025」这样的单独页面 |
| 关于页 | 放个人介绍、拍摄风格、联系方法 | 让别人知道你是谁 |

---

## 2. 这次先不做的内容

这些内容不是不能做，而是现在做会拖慢进度：

- Blog
- 站内搜索
- 邮件订阅
- 深色 / 浅色切换
- 多语言
- PWA
- 复杂动画
- 瀑布流布局

原因很简单：这些都不影响第一版上线。

---

## 3. 免费方案怎么选

这次采用一条最省事的免费路线：

| 项目 | 选择 | 大白话说明 |
| --- | --- | --- |
| 建站工具 | Astro | 帮你把网站页面生成出来。适合做静态网站。 |
| 代码托管 | GitHub | 相当于把网站源文件放到云端仓库里。 |
| 网站部署 | Vercel 免费版 | 相当于把网站正式挂到网上，让别人能打开。 |
| 域名 | 先用免费二级域名 | 先用 `xxx.vercel.app`，以后想买独立域名再加。 |

这条路线的优点：

- 免费可用
- 适合新手
- 后续扩展方便
- 不需要自己租服务器

---

## 4. 需要先理解的几个词

| 术语 | 大白话解释 |
| --- | --- |
| 静态网站 static site | 网站内容提前生成好。别人访问时直接看成品，所以速度快，维护也简单。 |
| 部署 deployment | 把你本地做好的网站放到网上。 |
| 仓库 repository | 存放网站文件的地方。可以理解为云端项目文件夹。 |
| 构建 build | 把源码整理成真正能上线的网站文件。 |
| 组件 component | 可重复使用的小页面模块，比如页头、页脚、照片卡片。 |
| 内容文件 content | 你写标题、描述、照片信息的地方。 |
| 响应式图片 responsive image | 根据手机或电脑屏幕大小，自动给出合适尺寸的图片，避免加载太大。 |

---

## 5. 第一步先准备什么

你只需要先准备这 4 类东西：

| 项目 | 要求 |
| --- | --- |
| 一台电脑 | 你现在这台 Mac 就可以 |
| Node.js | 用来运行建站工具 |
| GitHub 账号 | 用来保存网站项目 |
| 一批整理过的照片 | 先选 2 到 3 个系列，每个系列 6 到 12 张 |

建议先不要一次上传太多照片。第一版控制在：

- 2 到 3 个系列
- 每个系列 6 到 12 张
- 总数先控制在 20 到 30 张

这样更容易上线。

## 6. 网站内容结构先定死

先用下面这套内容结构，不再增加新页面。

### 6.1 首页

首页放这些内容：

1. 一张主照片
2. 你的名字
3. 一句简短介绍
4. 2 到 3 组精选作品入口
5. 一个进入 About 页的入口

首页示例文案结构：

```text
Leo
Photographer based in Hong Kong

关注街头、城市、光影与日常细节。
```

### 6.2 作品列表页

作品列表页每个系列显示：

- 系列封面
- 系列标题
- 拍摄日期
- 一句简介

### 6.3 单个作品系列页

每个系列页显示：

- 系列标题
- 简介
- 照片列表
- 可选的拍摄信息

### 6.4 关于页

关于页只放：

- 你是谁
- 你喜欢拍什么
- 常用器材
- 联系方式

## 7. 技术方案先简化

### 7.1 图片怎么放

网站里只放「展示版照片」，不放原图。

大白话解释：

- 原图太大，会让网站变慢
- 展示版是压缩后的版本，适合网页打开

建议做 3 种图片：

| 类型 | 建议尺寸 | 用途 |
| --- | --- | --- |
| 封面图 | 长边 1200px | 列表页和分享图 |
| 展示图 | 长边 1600px 到 2000px | 系列详情页 |
| 缩略图 | 长边 400px 到 600px | 小图预览 |

### 7.2 作品说明怎么存

每个作品系列配一个说明文件。

你可以把它理解为：

- 照片放在一个文件夹
- 标题、简介、日期写在另一个说明文件里

这样后面新增作品会很清楚。

### 7.3 布局怎么选

第一版只用普通网格布局。

大白话解释：

- 每张图整齐排开
- 电脑和手机都稳定
- 不容易出错

先不要用瀑布流。瀑布流看起来更花，但第一版更容易出布局问题。

## 8. 推荐的项目目录

```text
my-photo-site/
├── public/
│   └── favicon.svg
├── src/
│   ├── assets/
│   │   └── photos/
│   │       ├── hong-kong-street/
│   │       └── city-night/
│   ├── components/
│   │   ├── Header.astro
│   │   ├── Footer.astro
│   │   ├── PhotoGrid.astro
│   │   └── PhotoCard.astro
│   ├── content/
│   │   └── gallery/
│   │       ├── hong-kong-street.md
│   │       └── city-night.md
│   ├── layouts/
│   │   └── BaseLayout.astro
│   ├── pages/
│   │   ├── index.astro
│   │   ├── about.astro
│   │   └── gallery/
│   │       ├── index.astro
│   │       └── [slug].astro
│   └── content.config.ts
├── astro.config.mjs
└── package.json
```

你现在不用完全理解这些文件名。可以先把它理解成：

- `pages`：真正的网站页面
- `components`：重复使用的小模块
- `assets/photos`：网站里要展示的图片
- `content/gallery`：每个作品系列的文字说明

## 9. 实际操作流程

### 阶段 A：环境准备

目标：让电脑具备建站能力。

要做的事：

1. 安装 Node.js
2. 注册 GitHub
3. 确认电脑里有可用的照片文件夹

说明：

- 这一步不需要额外安装很多软件
- `Homebrew`、`nvm`、`ImageMagick` 这些先不列为必须项
- 先把网站跑起来，比先装一堆工具更重要

### 阶段 B：创建网站项目

目标：生成一个空白网站骨架。

要做的事：

1. 新建 Astro 项目
2. 安装必要依赖
3. 启动本地预览

大白话解释：

- 「本地预览」就是先在你自己电脑里看网站效果
- 这时网站还没正式上线

### 阶段 C：搭建 4 个页面

目标：把首页、作品页、关于页做出来。

顺序建议：

1. 先做统一页头和页脚
2. 再做首页
3. 再做作品列表页
4. 再做单个系列页
5. 最后做 About 页

### 阶段 D：整理照片和文字

目标：把真实内容放进去。

要做的事：

1. 每个系列选出照片
2. 给每个系列写标题和一句简介
3. 选一张封面图
4. 把照片压缩成适合网页展示的尺寸

### 阶段 E：免费上线

目标：让网站可以通过网址访问。

要做的事：

1. 把项目上传到 GitHub
2. 连接 Vercel
3. 一键发布
4. 获得一个免费网址

这个免费网址一般长这样：

```text
你的项目名.vercel.app
```

如果以后想要更正式的网址，再单独购买域名即可。

## 10. 第一版的页面要求

### 10.1 首页要求

- 顶部有名字和导航
- 首屏有一张主照片
- 有一句简短介绍
- 有精选系列入口

### 10.2 作品列表页要求

- 每个系列都有封面图
- 每个系列能点进去
- 手机上也能正常排版

### 10.3 单个系列页要求

- 照片能顺畅浏览
- 有标题和说明
- 页面加载不要太慢

### 10.4 关于页要求

- 简短介绍
- 联系方式
- 可选的器材信息

## 11. 零基础版本的时间预估

| 阶段 | 预计耗时 |
| --- | --- |
| 准备环境 | 半天 |
| 创建项目并跑起来 | 半天 |
| 搭建 4 个页面 | 1 到 2 天 |
| 整理照片和文案 | 1 天 |
| 免费上线 | 半天 |
| 合计 | 约 3 到 4 天 |

这个时间的前提是：

- 不做复杂动画
- 不做 Blog
- 不自己设计复杂 Logo
- 照片数量不超过 30 张

如果你一开始就追求很完整的视觉风格，时间会继续增加。

---

## 12. 第一版上线标准

满足这 6 条，就算第一版完成：

1. 网站能在网上打开
2. 手机和电脑都能正常看
3. 有首页、作品列表页、作品详情页、关于页
4. 至少有 2 个作品系列
5. 图片加载速度正常
6. 你自己愿意把网址发给别人

---

## 13. 现在最适合的执行顺序

如果按零基础思路推进，建议顺序固定如下：

1. 先搭空网站
2. 再把页面做出来
3. 再放真实照片
4. 最后再上线

不要一开始就纠结这些事：

- 要不要买域名
- 要不要做 Logo
- 要不要做 Blog
- 要不要做很多分类
- 要不要做复杂交互

这些都可以在网站上线之后再加。
